**Recursos identificados:**
- Usuarios (users)
- Restaurantes (restaurants)
- Menús (menus)
- Platos (dishes)
- Pedidos (orders)
- Pagos (payments)

Consideraciones:
- El seguimiento de pedidos se realiza por usuario y pedido. No es necesario un endpoint específico para el seguimiento, ya que se puede consultar el estado del pedido a través del endpoint de pedidos.
- Los restaurantes pueden actualizar el estado de los pedidos recibidos. Observa que, en lugar de tener múltiples endpoints para cada estado, se utiliza un único endpoint PATCH para actualizar el estado del pedido.
